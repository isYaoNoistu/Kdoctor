# Kdoctor 封版前最终优化文档

- 适用版本：`V2` 封版前最后一轮完善
- 目标：完善现有能力，不再继续扩展功能

| 项目 | 说明 |
| --- | --- |
| 文档定位 | 下一阶段开发与封版执行基线，以内部使用为目标，不以平台化扩展为目标 |
| 本轮硬约束 | 完全移除 JMX 相关目标；优先修正规则、证据、输出与语言一致性；完成后封版，仅保留极小范围缺陷修复 |

## 核心结论

从当前仓库与最近一轮运行输出看，`Kdoctor` 已经足以承担“内部 Kafka 首轮巡检 + 排障加速器”的角色。

封版前最后一轮工作只聚焦三件事：

1. 代码层规则优化
2. 输出格式优化
3. 参数与默认值收口

这轮不再新增功能域，不再继续把工具做大。

## 范围收口

### 保留

- 网络
- Kafka 元数据
- Topic / ISR
- Probe
- Compose
- Docker
- Host
- Logs
- 事务上下文

### 下线

- JMX / Metrics / JVM / Quota
- 依赖 JMX 的 Host / KRaft 路径
- 默认输出中与 JMX 相关的 `SKIP` 噪声

## P0 必须完成项

| 编号 | 优化方向 | 验收标准 |
| --- | --- | --- |
| P0-1 | 从 README、USER_GUIDE、示例 YAML、检查注册、摘要覆盖、输出模块名中移除 JMX/MET/JVM/QTA | 默认报告中不再出现相关检查和 `SKIP` |
| P0-2 | 采集覆盖改成三态：已启用、已获取证据、未纳入本次运行 | 摘要覆盖与检查明细不再互相打架 |
| P0-3 | `NET-003`、`NET-005`、`KFK-005` 证据去重 | 三种输出都不再明显重复 endpoint |
| P0-4 | 模块名、术语、句式统一中文 | 报告不再出现乱码和中英跳变 |
| P0-5 | `TOP-011` 只输出真正命中项 | 不再把正常 topic 一并写进告警证据 |

## P1 建议完成项

| 编号 | 优化方向 | 验收标准 |
| --- | --- | --- |
| P1-1 | 根因摘要继续做同源归并 | 同类问题不再拆成多条重复主因 |
| P1-2 | `LOG-001` 明确是日志来源与样本质量；`LOG-002` 只表示未命中已知指纹 | 日志模块每条结论都可解释 |
| P1-3 | 默认终端输出折叠 `PASS / SKIP` | 报告显著变短，但信息不丢 |
| P1-4 | 只在 compose / logs / docker / host 明确可用时才注册相关路径 | 无关检查数量下降 |
| P1-5 | 统一编号 + 模块中文名 + 摘要格式 | 终端和 Markdown 扫读体验一致 |

## 输出收口标准

默认终端报告只应重点回答三件事：

1. 有没有真故障
2. 最值得先看什么
3. 哪些只是上下文提示

### 默认终端输出要求

- 保留页头摘要、主因判断、建议动作、重点问题
- `PASS / SKIP` 默认不逐条打印
- 只展开 `FAIL / WARN / ERROR / CRIT`
- 证据最多展示前 `N` 条，超出后显示“其余 X 条已省略”

### Markdown 输出要求

- 不是终端内容平铺
- 需要有表格化摘要
- 需要有重点问题汇总
- 需要有可展开的完整附录

## 参数默认值建议

| 参数 | 建议默认值 |
| --- | --- |
| `logs.lookback_minutes` | `30` |
| `logs.tail_lines` | `400~800`，当前收口为 `500` |
| `probe.topic` | `_kdoctor_probe` |
| `probe.message_bytes` | `1024` |
| `summary.max_root_causes` | `3` |
| `summary.max_actions` | `5` |
| `output.max_evidence_items` | `8` |
| `output.show_pass_checks` | `false` |
| `output.show_skip_checks` | `false` |
| `output.verbose` | `false` |

## 封版验收标准

满足以下五条即可封版：

1. 默认终端报告中不再出现任何 JMX / MET / JVM / QTA 相关检查
2. 摘要覆盖与明细检查不再相互矛盾
3. `TOP-011`、`NET-003/005`、日志模块的证据不再误导或重复
4. 报告语言统一为中文，无乱码、无显著中英混排
5. 三套固定样例的终端与 Markdown 输出通过人工验收

## 最终结论

`Kdoctor` 的最后阶段不需要再做大，而是要把已有能力磨到“更稳、更准、更短、更像给值班人看的报告”。

对当前内部场景最重要的决策是：

- 去掉无法启用的 JMX 路径
- 统一证据语义
- 减少误导与噪声
- 优化输出格式

只要 P0 全部完成、P1 完成大部分、封版验收通过，就可以停止继续扩功能，正式投入内部使用。
